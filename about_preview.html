?? How Aegis Protocol Works

Our agentic AI system uses three specialized agents working together to verify claims and detect misinformation.

Agentic AI Architecture

Aegis Protocol employs a multi-agent system where each AI agent has a specific role in the verification pipeline. When you submit a claim, it passes through three intelligent agents that collaborate to determine its veracity.

??

Agent 1: Claim Ingestion Agent

Role: Normalizes and prepares claims for processing

What It Does:

  • Text Normalization: Standardizes the claim text by removing extra whitespace, converting to lowercase, and cleaning formatting
  • Duplicate Detection: Generates a SHA256 hash to identify duplicate claims and avoid redundant processing
  • Structured Output: Returns a JSON response with the normalized text and unique claim hash

Technology: Pure Python text processing with cryptographic hashing

??

Agent 2: Research Agent

Role: Gathers evidence both supporting and refuting the claim

What It Does:

  • Evidence Gathering: Uses Google Gemini AI to research and compile factual evidence related to the claim
  • Balanced Analysis: Collects both supporting evidence (facts that confirm the claim) and refuting evidence (facts that contradict it)
  • Source Quality: Prioritizes credible, verifiable information from reliable sources
  • Structured Evidence: Returns organized JSON with separate lists of supporting and refuting evidence points

Technology: Google Gemini 2.0 Flash Lite with advanced prompt engineering for fact-checking

??

Agent 3: Investigator Agent

Role: Analyzes evidence and determines the final verdict

What It Does:

  • Evidence Analysis: Evaluates all supporting and refuting evidence collected by the Research Agent
  • Verdict Determination: Determines if the claim is True, False, or Misleading based on the weight of evidence
  • Confidence Scoring: Assigns a confidence score (0-100) indicating how certain the verdict is
  • Severity Assessment: Rates the potential harm level (low, medium, high, critical) if the claim is false or misleading
  • Reasoning Explanation: Provides a detailed explanation of how the verdict was reached

Technology: Google Gemini 2.0 Flash Lite with high-reasoning prompts for critical analysis

?? The Complete Processing Pipeline

  1. Submission: User submits a claim via the Submit Claim page
  2. Ingestion: Claim Ingestion Agent normalizes and hashes the claim
  3. Database Storage: Claim is saved to Supabase with status "pending"
  4. Background Processing: Claim worker picks up the pending claim
  5. Research Phase: Research Agent gathers supporting and refuting evidence
  6. Investigation Phase: Investigator Agent analyzes evidence and determines verdict
  7. Result Storage: Final verdict, confidence, severity, and reasoning are saved
  8. Status Update: Claim status updated to "completed" or "failed"

?? Why Agentic AI?

Specialization: Each agent focuses on a specific task, leading to better performance than a single monolithic system.

Scalability: Agents can be improved or replaced independently without affecting the entire system.

Transparency: The multi-step process makes it easier to understand how verdicts are reached.

Reliability: If one agent encounters an issue, the system can gracefully handle errors and provide feedback.
