// SYSTEM ARCHITECTURE

How Aegis Protocol Works

A three-agent pipeline built for speed, accuracy and transparency — from claim submission to final verdict in seconds.

// PIPELINE
01

Submission

User submits a claim via the Submit Claim page. The system accepts free-form natural language — no special formatting needed.

02

Ingestion Agent — Normalize & Hash

Standardizes claim text, strips noise, generates a SHA256 hash to detect and skip duplicate claims before processing.

03

Database Storage

Claim is saved to Supabase (PostgreSQL) with status pending — immediately queryable from the dashboard.

04

Research Agent — Evidence Gathering

Uses Gemini AI to compile both supporting and refuting evidence from credible sources. Returns structured JSON with separate evidence lists.

05

Investigator Agent — Verdict

Analyzes all evidence, determines True / False / Misleading, assigns a confidence score (0–100) and severity rating (low → critical).

06

Result Storage & Status Update

Verdict, confidence, severity and reasoning are saved. Claim status flips to completed. Dashboard updates in real-time.

// THE THREE CORE AGENTS
AGENT_01
Ingestion Agent

Normalizes text, deduplicates via SHA256 hash, and packages the claim for downstream agents. Pure Python — fast and deterministic.

AGENT_02
Research Agent

Queries Gemini 2.5 Flash to gather balanced evidence — both supporting and refuting the claim — from reliable, verifiable sources.

AGENT_03
Investigator Agent

Weighs all evidence, delivers a final True / False / Misleading verdict with confidence score, severity rating, and detailed reasoning.

// WHY AGENTIC AI
Specialization

Each agent focuses on a specific task — ingestion, research, or investigation — leading to higher quality than a single monolithic model.

Scalability

Agents can be upgraded or swapped independently. Swap research sources without touching the verdict logic.

Transparency

Multi-step processing makes reasoning auditable. Every verdict includes the evidence that led to it.

Reliability

If one agent encounters an issue, the system degrades gracefully rather than failing silently.
